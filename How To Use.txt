SVANIPP — HOW TO USE

This guide explains everything you need, from setup to sending your first file.
No technical knowledge required.

Quick setup if you just cloned this repo on Windows:

PowerShell:
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\quickstart-windows.ps1

This installs dependencies, builds, and runs a readiness check.

────────────────────────
PART 1 — WHAT YOU NEED FIRST
────────────────────────

Before you can use Svanipp, your computer must have:

✔ A C++ compiler
✔ CMake
✔ The Svanipp source code
✔ Two devices on the same network (Wi-Fi or LAN)

If you are using Windows, the easiest setup is:

Install MSYS2 (includes GCC compiler)

Install CMake

Install Visual Studio Code (optional but helpful)

If you already built svanipp.exe, you are ready to use it.

────────────────────────
PART 2 — BUILD THE PROGRAM
────────────────────────

If you only have the source code, build it first.

Open a terminal in the Svanipp folder and run:

Windows (recommended):

.\scripts\build-windows.ps1 -BuildType Release

or manual CMake:

cmake -S . -B build
cmake --build build

After this, you will see:

build/svanipp.exe

That is the program you will run.

> If you use Visual Studio or the CMake Tools extension, the executable may appear in `out\build\x64-Debug\svanipp.exe` for debug builds. For the scripts in this repository, use `build\svanipp.exe`.

────────────────────────
PART 3 — BASIC IDEA
────────────────────────

Svanipp sends files between devices on the same network.

Think of it like handing a book to a friend sitting next to you.

One computer RECEIVES.
One computer SENDS.

────────────────────────
PART 4 — START THE RECEIVER
────────────────────────

On the computer that will receive the file:

Open PowerShell or Terminal and run:

build\svanipp.exe receive --port 39000 --out Downloads

What this means:
• The computer waits for incoming files
• Files will be saved in the "Downloads" folder

Leave this window open.

────────────────────────
PART 5 — FIND OTHER DEVICES
────────────────────────

On the sending computer, you can see available devices:

build\svanipp.exe discover

Example output:

192.168.1.10  MY-PC  39000

This means the device MY-PC is ready to receive files.

────────────────────────
PART 6 — SEND A FILE
────────────────────────

You can send using the IP address:

svanipp send --ip 192.168.1.10 myfile.txt

Or using the device name:

svanipp send --name MY-PC myfile.txt

What happens:
• The file is transferred
• You see progress on both computers
• The receiver saves the file

────────────────────────
PART 7 — WHAT YOU WILL SEE
────────────────────────

Sender:
Sending myfile.txt
  100%
Sent: myfile.txt (0.25 MB)

Receiver:
Receiving myfile.txt ... 100%
Saved: Downloads\myfile.txt (0.25 MB)

Files arrive safely: each file is verified with SHA-256. Only errors are reported.

────────────────────────
PART 8 — SENDING THE SAME FILE AGAIN
────────────────────────

If you send the same file a second time:

By default, Svanipp will ASK if you want to overwrite:

  File exists: myfile.txt. Overwrite? (y/n): 

Type "y" or "n":
• y = replace the old file
• n = keep the old file; new one will be named myfile (1).txt

If you want automatic overwrite without prompting, start receiver with:

svanipp receive --port 39000 --out Downloads --overwrite

This skips the prompt and always overwrites without asking.

────────────────────────
PART 9 — TROUBLESHOOTING
────────────────────────

Nothing appears in discover?
✔ Make sure both devices are on the same network
✔ Make sure the receiver is running

Connection failed?
✔ Check the IP address
✔ Ensure firewall allows the program

File not received?
✔ Make sure the receiver window is still open

────────────────────────
PART 10 — QUICK HEALTH CHECK (RECOMMENDED)
────────────────────────

Before demos or transfers, run this once:

svanipp check

This command quickly verifies:
✔ Version
✔ Local IP address
✔ Hostname
✔ Transfer port
✔ Discovery port
✔ Terminal support (TTY, colors, TUI auto)
✔ Bind test on transfer port

You will get a final status:
• READY
or
• ISSUES FOUND

If bind test fails with "port in use", close the process using that port
or choose a different transfer port and test it directly:

svanipp check --port 39001

Exit code:
• 0 = ready
• non-zero = issues found
